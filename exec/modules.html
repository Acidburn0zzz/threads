Modules¶

External Typing¶

For the purpose of checking external values against imports, such values are classified by external types. The following auxiliary typing rules specify this typing relation relative to a store \(S\) in which the external value lives.

\(\mathsf{func}~a\)¶

  • The store entry \(S.\mathsf{funcs}[a]\) must be a function instance \(\{\mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \dots\}\).
  • Then \(\mathsf{func}~a\) is valid with external type \(\mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}\).
\[\frac{ S.\mathsf{funcs}[a] = \{\mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \dots\} }{ S \vdash \mathsf{func}~a : \mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}} }\]

\(\mathsf{table}~a\)¶

  • The store entry \(S.\mathsf{tables}[a]\) must be a table instance \(\{\mathsf{elem}~(\mathit{fa}^?)^n, \mathsf{max}~m^?\}\).
  • Then \(\mathsf{table}~a\) is valid with external type \(\mathsf{table}~(\{\mathsf{min}~n, \mathsf{max}~m^?\}~\href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}})\).
\[\frac{ S.\mathsf{tables}[a] = \{ \mathsf{elem}~(\mathit{fa}^?)^n, \mathsf{max}~m^? \} }{ S \vdash \mathsf{table}~a : \mathsf{table}~(\{\mathsf{min}~n, \mathsf{max}~m^?\}~\href{../syntax/types.html#syntax-elemtype}{\mathsf{anyfunc}}) }\]

\(\mathsf{mem}~a\)¶

  • The store entry \(S.\mathsf{mems}[a]\) must be a memory instance \(\{\mathsf{data}~b^{n\cdot64\,\mathrm{Ki}}, \mathsf{max}~m^?\}\), for some \(n\).
  • Then \(\mathsf{mem}~a\) is valid with external type \(\mathsf{mem}~(\{\mathsf{min}~n, \mathsf{max}~m^?\})\).
\[\frac{ S.\mathsf{mems}[a] = \{ \mathsf{data}~b^{n\cdot64\,\mathrm{Ki}}, \mathsf{max}~m^? \} }{ S \vdash \mathsf{mem}~a : \mathsf{mem}~\{\mathsf{min}~n, \mathsf{max}~m^?\} }\]

\(\mathsf{global}~a\)¶

  • The store entry \(S.\mathsf{globals}[a]\) must be a global instance \(\{\mathsf{value}~(t.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~c), \mathsf{mut}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}}\}\).
  • Then \(\mathsf{global}~a\) is valid with external type \(\mathsf{global}~(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t)\).
\[\frac{ S.\mathsf{globals}[a] = \{ \mathsf{value}~(t.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~c), \mathsf{mut}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}} \} }{ S \vdash \mathsf{global}~a : \mathsf{global}~(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t) }\]

Import Matching¶

When instantiating a module, external values must be provided whose types are matched against the respective external types classifying each import. In some cases, this allows for a simple form of subtyping, as defined below.

Limits¶

Limits \(\{ \mathsf{min}~n_1, \mathsf{max}~m_1^? \}\) match limits \(\{ \mathsf{min}~n_2, \mathsf{max}~m_2^? \}\) if and only if:

  • \(n_1\) is not smaller than \(n_2\).
  • Either:
    • \(m_2^?\) is empty.
  • Or:
    • Both \(m_1^?\) and \(m_2^?\) are non-empty.
    • \(m_1\) is not larger than \(m_2\).
\[\frac{ n_1 \geq n_2 }{ \vdash \{ \mathsf{min}~n_1, \mathsf{max}~m_1^? \} \leq \{ \mathsf{min}~n_2, \mathsf{max}~\epsilon \} } \quad \frac{ n_1 \geq n_2 \qquad m_1 \leq m_2 }{ \vdash \{ \mathsf{min}~n_1, \mathsf{max}~m_1 \} \leq \{ \mathsf{min}~n_2, \mathsf{max}~m_2 \} }\]

Functions¶

An external type \(\mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_1\) matches \(\mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_2\) if and only if:

  • Both \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_1\) and \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}_2\) are the same.
\[\frac{ }{ \vdash \mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}} \leq \mathsf{func}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}} }\]

Tables¶

An external type \(\mathsf{table}~(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}_1)\) matches \(\mathsf{table}~(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}_2)\) if and only if:

  • Limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1\) match \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2\).
  • Both \(\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}_1\) and \(\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}_2\) are the same.
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1 \leq \href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2 }{ \vdash \mathsf{table}~(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}) \leq \mathsf{table}~(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}}) }\]

Memories¶

An external type \(\mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1\) matches \(\mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2\) if and only if:

  • Limits \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1\) match \(\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2\).
\[\frac{ \vdash \href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1 \leq \href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2 }{ \vdash \mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_1 \leq \mathsf{mem}~\href{../syntax/types.html#syntax-limits}{\mathit{limits}}_2 }\]

Globals¶

An external type \(\mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_1\) matches \(\mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_2\) if and only if:

  • Both \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_1\) and \(\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}}_2\) are the same.
\[\frac{ }{ \vdash \mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} \leq \mathsf{global}~\href{../syntax/types.html#syntax-globaltype}{\mathit{globaltype}} }\]

Allocation¶

New instances of functions, tables, memories, globals, and modules are allocated in a store \(S\), as defined by the following auxiliary functions.

Functions¶

  1. Let \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}\) be the function to allocate and \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\) its module instance.
  2. Let \(a\) be the first free function address in \(S\).
  3. Let \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}\) be the function type \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{types}[\href{../syntax/modules.html#syntax-func}{\mathit{func}}.\mathsf{type}]\).
  4. Let \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\) be the function instance \(\{ \mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \mathsf{module}~\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}, \mathsf{code}~\href{../syntax/modules.html#syntax-func}{\mathit{func}} \}\).
  5. Append \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\) to the \(\mathsf{funcs}\) of \(S\).
  6. Return \(a\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-func}{\mathrm{allocfunc}}(S, \href{../syntax/modules.html#syntax-func}{\mathit{func}}, \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}) &=& S', \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}} \\[1ex] \mbox{where:} \hfill \\ \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}} &=& |S.\mathsf{funcs}| \\ \href{../syntax/types.html#syntax-functype}{\mathit{functype}} &=& \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{types}[\href{../syntax/modules.html#syntax-func}{\mathit{func}}.\mathsf{type}] \\ \href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}} &=& \{ \mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \mathsf{module}~\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}, \mathsf{code}~\href{../syntax/modules.html#syntax-func}{\mathit{func}} \} \\ S' &=& S \href{../syntax/conventions.html#syntax-record}{\oplus} \{\mathsf{funcs}~\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\} \\ \end{array}\end{split}\]

Host Functions¶

  1. Let \(\href{../exec/runtime.html#syntax-hostfunc}{\mathit{hostfunc}}\) be the host function to allocate and \(\href{../syntax/types.html#syntax-functype}{\mathit{functype}}\) its function type.
  2. Let \(a\) be the first free function address in \(S\).
  1. Let \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\) be the function instance \(\{ \mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \mathsf{code}~\href{../exec/runtime.html#syntax-hostfunc}{\mathit{hostfunc}} \}\).
  2. Append \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\) to the \(\mathsf{funcs}\) of \(S\).
  3. Return \(a\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-hostfunc}{\mathrm{allochostfunc}}(S, \href{../exec/runtime.html#syntax-hostfunc}{\mathit{hostfunc}}, \href{../syntax/types.html#syntax-functype}{\mathit{functype}}) &=& S', \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}} \\[1ex] \mbox{where:} \hfill \\ \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}} &=& |S.\mathsf{funcs}| \\ \href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}} &=& \{ \mathsf{type}~\href{../syntax/types.html#syntax-functype}{\mathit{functype}}, \mathsf{code}~\href{../exec/runtime.html#syntax-hostfunc}{\mathit{hostfunc}} \} \\ S' &=& S \href{../syntax/conventions.html#syntax-record}{\oplus} \{\mathsf{funcs}~\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}\} \\ \end{array}\end{split}\]

Note

Host functions are never allocated by the WebAssembly semantics itself, but may be allocated by the embedder.

Tables¶

  1. Let \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}\) be the table to allocate.
  2. Let \((\{\mathsf{min}~n, \mathsf{max}~m^?\}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}})\) be the table type \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}.\mathsf{type}\).
  3. Let \(a\) be the first free table address in \(S\).
  4. Let \(\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}\) be the table instance \(\{ \mathsf{elem}~(\epsilon)^n, \mathsf{max}~m^? \}\) with \(n\) empty elements.
  5. Append \(\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}\) to the \(\mathsf{tables}\) of \(S\).
  6. Return \(a\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-table}{\mathrm{alloctable}}(S, \href{../syntax/modules.html#syntax-table}{\mathit{table}}) &=& S', \href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}} \\[1ex] \mbox{where:} \hfill \\ \href{../syntax/modules.html#syntax-table}{\mathit{table}}.\mathsf{type} &=& \{\mathsf{min}~n, \mathsf{max}~m^?\}~\href{../syntax/types.html#syntax-elemtype}{\mathit{elemtype}} \\ \href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}} &=& |S.\mathsf{tables}| \\ \href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}} &=& \{ \mathsf{elem}~(\epsilon)^n, \mathsf{max}~m^? \} \\ S' &=& S \href{../syntax/conventions.html#syntax-record}{\oplus} \{\mathsf{tables}~\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}\} \\ \end{array}\end{split}\]

Memories¶

  1. Let \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}\) be the memory to allocate.
  2. Let \(\{\mathsf{min}~n, \mathsf{max}~m^?\}\) be the table type \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}.\mathsf{type}\).
  3. Let \(a\) be the first free memory address in \(S\).
  4. Let \(\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}\) be the memory instance \(\{ \mathsf{data}~(\def\mathdef1082#1{\mathtt{0x#1}}\mathdef1082{00})^{n \cdot 64\,\mathrm{Ki}}, \mathsf{max}~m^? \}\) that contains \(n\) pages of zeroed bytes.
  5. Append \(\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}\) to the \(\mathsf{mems}\) of \(S\).
  6. Return \(a\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-mem}{\mathrm{allocmem}}(S, \href{../syntax/modules.html#syntax-mem}{\mathit{mem}}) &=& S', \href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}} \\[1ex] \mbox{where:} \hfill \\ \href{../syntax/modules.html#syntax-mem}{\mathit{mem}}.\mathsf{type} &=& \{\mathsf{min}~n, \mathsf{max}~m^?\} \\ \href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}} &=& |S.\mathsf{mems}| \\ \href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}} &=& \{ \mathsf{data}~(\def\mathdef1083#1{\mathtt{0x#1}}\mathdef1083{00})^{n \cdot 64\,\mathrm{Ki}}, \mathsf{max}~m^? \} \\ S' &=& S \href{../syntax/conventions.html#syntax-record}{\oplus} \{\mathsf{mems}~\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}\} \\ \end{array}\end{split}\]

Globals¶

  1. Let \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}\) be the global to allocate.
  2. Let \(\href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t\) be the global type \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}.\mathsf{type}\).
  3. Let \(a\) be the first free global address in \(S\).
  4. Let \(\href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}}\) be the global instance \(\{ \mathsf{value}~(t.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~0), \mathsf{mut}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}} \}\) whose contents is a zero value of value type \(t\).
  5. Append \(\href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}}\) to the \(\mathsf{globals}\) of \(S\).
  6. Return \(a\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-global}{\mathrm{allocglobal}}(S, \href{../syntax/modules.html#syntax-global}{\mathit{global}}) &=& S', \href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}} \\[1ex] \mbox{where:} \hfill \\ \href{../syntax/modules.html#syntax-global}{\mathit{global}}.\mathsf{type} &=& \href{../syntax/types.html#syntax-mut}{\mathit{mut}}~t \\ \href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}} &=& |S.\mathsf{globals}| \\ \href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}} &=& \{ \mathsf{value}~(t.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~0), \mathsf{mut}~\href{../syntax/types.html#syntax-mut}{\mathit{mut}} \} \\ S' &=& S \href{../syntax/conventions.html#syntax-record}{\oplus} \{\mathsf{globals}~\href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}}\} \\ \end{array}\end{split}\]

Modules¶

The allocation function for modules requires a suitable list of external values that are assumed to match the import vector of the module.

  1. Let \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) be the module to allocate and \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast\) the vector of external values providing the module’s imports.
  2. For each function \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{funcs}\), do:
    1. Let \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_i\) be the function address resulting from allocating \(\href{../syntax/modules.html#syntax-func}{\mathit{func}}_i\) for the module instance \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\) defined below.
  3. For each table \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{tables}\), do:
    1. Let \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_i\) be the table address resulting from allocating \(\href{../syntax/modules.html#syntax-table}{\mathit{table}}_i\).
  4. For each memory \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{mems}\), do:
    1. Let \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_i\) be the memory address resulting from allocating \(\href{../syntax/modules.html#syntax-mem}{\mathit{mem}}_i\).
  5. For each global \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals}\), do:
    1. Let \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_i\) be the global address resulting from allocating \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\).
  6. Let \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}^\ast\) be the the concatenation of the function addresses \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_i\) in index order.
  7. Let \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}^\ast\) be the the concatenation of the table addresses \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_i\) in index order.
  8. Let \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}^\ast\) be the the concatenation of the memory addresses \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_i\) in index order.
  9. Let \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}^\ast\) be the the concatenation of the global addresses \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_i\) in index order.
  10. Let \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_{\mathrm{mod}}^\ast\) be the list of function addresses extracted from \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast\), concatenated with \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}^\ast\).
  11. Let \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_{\mathrm{mod}}^\ast\) be the list of table addresses extracted from \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast\), concatenated with \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}^\ast\).
  12. Let \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_{\mathrm{mod}}^\ast\) be the list of memory addresses extracted from \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast\), concatenated with \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}^\ast\).
  13. Let \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_{\mathrm{mod}}^\ast\) be the list of global addresses extracted from \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast\), concatenated with \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}^\ast\).
  14. For each export \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports}\), do:
    1. If \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) is a function export for function index \(x\), then let \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) be the external value \(\mathsf{func}~(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_{\mathrm{mod}}^\ast[x])\).
    2. Else, if \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) is a table export for table index \(x\), then let \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) be the external value \(\mathsf{table}~(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_{\mathrm{mod}}^\ast[x])\).
    3. Else, if \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) is a memory export for memory index \(x\), then let \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) be the external value \(\mathsf{mem}~(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_{\mathrm{mod}}^\ast[x])\).
    4. Else, if \(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i\) is a global export for global index \(x\), then let \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) be the external value \(\mathsf{global}~(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_{\mathrm{mod}}^\ast[x])\).
    5. Let \(\href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}_i\) be the export instance \(\{\mathsf{name}~(\href{../syntax/modules.html#syntax-export}{\mathit{export}}_i.\mathsf{name}), \mathsf{value}~\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\}\).
  15. Let \(\href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}^\ast\) be the the concatenation of the export instances \(\href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}_i\) in index order.
  16. Let \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\) be the module instance \(\{\mathsf{types}~(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{types}),\) \(\mathsf{funcs}~\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_{\mathrm{mod}}^\ast,\) \(\mathsf{tables}~\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_{\mathrm{mod}}^\ast,\) \(\mathsf{mems}~\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_{\mathrm{mod}}^\ast,\) \(\mathsf{globals}~\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_{\mathrm{mod}}^\ast,\) \(\mathsf{exports}~\href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}^\ast\}\).
  17. Return \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\).
\[\begin{split}\begin{array}{rlll} \href{../exec/modules.html#alloc-module}{\mathrm{allocmodule}}(S, \href{../syntax/modules.html#syntax-module}{\mathit{module}}, \href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast) &=& S', \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}} \\[1ex] \mbox{where:} \hfill \\ \end{array}\end{split}\]
\[\begin{split}\begin{array}{rlll} \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}} &=& \{~ \begin{array}[t]{@{}l@{}} \mathsf{types}~\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{types}, \\ \mathsf{funcs}~\href{../syntax/types.html#syntax-externtype}{\mathrm{funcs}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast)~\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}^\ast, \\ \mathsf{tables}~\href{../syntax/types.html#syntax-externtype}{\mathrm{tables}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast)~\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}^\ast, \\ \mathsf{mems}~\href{../syntax/types.html#syntax-externtype}{\mathrm{mems}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast)~\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}^\ast, \\ \mathsf{globals}~\href{../syntax/types.html#syntax-externtype}{\mathrm{globals}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{im}}^\ast)~\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}^\ast, \\ \mathsf{exports}~\href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}^\ast ~\} \end{array} \\[1ex] S_1, \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}^\ast &=& \href{../exec/modules.html#alloc-func}{\mathrm{allocfunc}}^\ast(S, \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{funcs}, \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}) \\ S_2, \href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}^\ast &=& \href{../exec/modules.html#alloc-table}{\mathrm{alloctable}}^\ast(S_1, \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{tables}) \\ S_3, \href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}^\ast &=& \href{../exec/modules.html#alloc-mem}{\mathrm{allocmem}}^\ast(S_2, \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{mems}) \\ S', \href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}^\ast &=& \href{../exec/modules.html#alloc-global}{\mathrm{allocglobal}}^\ast(S_3, \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals}) \\[1ex] \href{../exec/runtime.html#syntax-exportinst}{\mathit{exportinst}}^\ast &=& \{ \mathsf{name}~(\href{../syntax/modules.html#syntax-export}{\mathit{export}}.\mathsf{name}), \mathsf{value}~\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{ex}} \}^\ast & (\href{../syntax/modules.html#syntax-export}{\mathit{export}}^\ast = \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports}) \\ \href{../syntax/types.html#syntax-externtype}{\mathrm{funcs}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{ex}}^\ast) &=& (\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{funcs}[x])^\ast & (x^\ast = \href{../syntax/types.html#syntax-externtype}{\mathrm{funcs}}(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports})) \\ \href{../syntax/types.html#syntax-externtype}{\mathrm{tables}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{ex}}^\ast) &=& (\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{tables}[x])^\ast & (x^\ast = \href{../syntax/types.html#syntax-externtype}{\mathrm{tables}}(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports})) \\ \href{../syntax/types.html#syntax-externtype}{\mathrm{mems}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{ex}}^\ast) &=& (\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{mems}[x])^\ast & (x^\ast = \href{../syntax/types.html#syntax-externtype}{\mathrm{mems}}(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports})) \\ \href{../syntax/types.html#syntax-externtype}{\mathrm{globals}}(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_{\mathrm{ex}}^\ast) &=& (\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{globals}[x])^\ast & (x^\ast = \href{../syntax/types.html#syntax-externtype}{\mathrm{globals}}(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{exports})) \\ \end{array}\end{split}\]

Here, the notation \(\mathrm{allocX}^\ast\) is shorthand for multiple allocations of object kind \(X\), defined as follows:

\[\begin{split}\begin{array}{rlll} \mathrm{allocX}^\ast(S_0, X^n, \dots) &=& S_n, a^n \\[1ex] \mbox{where for all $i < n$:} \hfill \\ S_{i+1}, a^n[i] &=& \mathrm{allocX}(S_i, X^n[i], \dots) \end{array}\end{split}\]

Note

The definition of module allocation is mutually recursive with the allocation of its associated functions, because the resulting module instance \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\) is passed to the function allocator as an argument, in order to form the necessary closures. In an implementation, this recursion is easily unraveled by mutating one or the other in a secondary step.

Instantiation¶

Given a store \(S\), a module \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) is instantiated with a list of external values \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}^n\) supplying the required imports as follows. Instantiation may fail with an error.

  1. If \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) is not valid, then:
    1. Fail.
  2. Assert: \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) is valid with external types \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^m\) classifying its imports.
  3. If the number \(m\) of imports is not equal to the number \(n\) of provided external values, then:
    1. Fail.
  4. For each external value \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) in \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}^n\) and external type \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}_i\) in \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^n\), do:
    1. Assert: \(\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}_i\) is valid with external type \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}'_i\) in store \(S\).
    2. If \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}'_i\) does not match \(\href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}_i\), then:
      1. Fail.
  5. Let \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}\) be a new module instance allocated from \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}\) in store \(S\).
  6. Let \(F\) be the frame \(\{ \mathsf{module}~\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}, \mathsf{locals}~\epsilon \}\).
  7. Push the frame \(F\) to the stack.
  8. For each element segment \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{elem}\), do:
    1. Let \(\mathit{eoval}_i\) be the result of evaluating the expression \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i.\mathsf{offset}\).
    2. Assert: due to validation, \(\mathit{eoval}_i\) is of the form \(\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~\mathit{eo}_i\).
    3. Let \(\href{../syntax/modules.html#syntax-tableidx}{\mathit{tableidx}}_i\) be the table index \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i.\mathsf{table}\).
    4. Assert: due to validation, \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{tables}[\href{../syntax/modules.html#syntax-tableidx}{\mathit{tableidx}}_i]\) exists.
    5. Let \(\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_i\) be the table address \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{tables}[\href{../syntax/modules.html#syntax-tableidx}{\mathit{tableidx}}_i]\).
    6. Assert: due to validation, \(S.\mathsf{tables}[\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_i]\) exists.
    7. Let \(\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}_i\) be the table instance \(S.\mathsf{tables}[\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}_i]\).
    8. Let \(\mathit{eend}_i\) be \(\mathit{eo}_i\) plus the length of \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i.\mathsf{init}\).
    9. If \(\mathit{eend}_i\) is larger than the length of \(\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}_i.\mathsf{elem}\), then:
      1. Fail.
  9. For each data segment \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{data}\), do:
    1. Let \(\mathit{doval}_i\) be the result of evaluating the expression \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i.\mathsf{offset}\).
    2. Assert: due to validation, \(\mathit{doval}_i\) is of the form \(\href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~\mathit{do}_i\).
    3. Let \(\href{../syntax/modules.html#syntax-memidx}{\mathit{memidx}}_i\) be the memory index \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i.\mathsf{mem}\).
    4. Assert: due to validation, \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{mems}[\href{../syntax/modules.html#syntax-memidx}{\mathit{memidx}}_i]\) exists.
    5. Let \(\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_i\) be the memory address \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{mems}[\href{../syntax/modules.html#syntax-memidx}{\mathit{memidx}}_i]\).
    6. Assert: due to validation, \(S.\mathsf{mems}[\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_i]\) exists.
    7. Let \(\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}_i\) be the memory instance \(S.\mathsf{mems}[\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}_i]\).
    8. Let \(\mathit{dend}_i\) be \(\mathit{do}_i\) plus the length of \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i.\mathsf{init}\).
    9. If \(\mathit{dend}_i\) is larger than the length of \(\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}_i.\mathsf{data}\), then:
      1. Fail.
  10. Let \(\href{../syntax/modules.html#syntax-globalidx}{\mathit{globalidx}}_{\mathrm{new}}\) be the global index that corresponds to the number of global imports in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{imports}\) (i.e., the index of the first non-imported global).
  11. For each global \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals}\), do:
    1. Let \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_i\) be the result of evaluating the initializer expression \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i.\mathsf{init}\).
    2. Let \(\href{../syntax/modules.html#syntax-globalidx}{\mathit{globalidx}}_i\) be the global index \(\href{../syntax/modules.html#syntax-globalidx}{\mathit{globalidx}}_{\mathrm{new}} + i\).
    3. Assert: due to validation, \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{globals}[\href{../syntax/modules.html#syntax-globalidx}{\mathit{globalidx}}_i]\) exists.
    4. Let \(\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_i\) be the global address \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{globals}[\href{../syntax/modules.html#syntax-globalidx}{\mathit{globalidx}}_i]\).
    5. Assert: due to validation, \(S.\mathsf{globals}[\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_i]\) exists.
    6. Let \(\href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}}_i\) be the global instance \(S.\mathsf{globals}[\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}_i]\).
  12. Assert: due to validation, the frame \(F\) is now on the top of the stack.
  13. Pop the frame from the stack.
  14. For each element segment \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{elem}\), do:
    1. For each function index \(\href{../syntax/modules.html#syntax-funcidx}{\mathit{funcidx}}_{ij}\) in \(\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}_i.\mathsf{init}\) (starting with \(j = 0\)), do:
      1. Assert: due to validation, \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{funcs}[\href{../syntax/modules.html#syntax-funcidx}{\mathit{funcidx}}_{ij}]\) exists.
      2. Let \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_{ij}\) be the function address \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{funcs}[\href{../syntax/modules.html#syntax-funcidx}{\mathit{funcidx}}_{ij}]\).
      3. Replace \(\href{../exec/runtime.html#syntax-tableinst}{\mathit{tableinst}}_i.\mathsf{elem}[\mathit{eo}_i + j]\) with \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}_{ij}\).
  15. For each data segment \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{data}\), do:
    1. For each byte \(b_{ij}\) in \(\href{../syntax/modules.html#syntax-data}{\mathit{data}}_i.\mathsf{init}\) (starting with \(j = 0\)), do:
      1. Replace \(\href{../exec/runtime.html#syntax-meminst}{\mathit{meminst}}_i.\mathsf{data}[\mathit{do}_i + j]\) with \(b_{ij}\).
  16. For each global \(\href{../syntax/modules.html#syntax-global}{\mathit{global}}_i\) in \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals}\), do:
    1. Replace \(\href{../exec/runtime.html#syntax-globalinst}{\mathit{globalinst}}_i.\mathsf{value}\) with \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_i\).
  17. If the start function \(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start}\) is not empty, then:
    1. Assert: due to validation, \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{funcs}[\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start}]\) exists.
    2. Let \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}\) be the function address \(\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{funcs}[\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start}]\).
    3. Invoke the function instance at \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}\).
\[\begin{split}\begin{array}{@{}rcll} S; \href{../exec/runtime.html#syntax-instantiate}{\mathsf{instantiate}}~\href{../syntax/modules.html#syntax-module}{\mathit{module}}~\href{../exec/runtime.html#syntax-externval}{\mathit{externval}}^n &\hookrightarrow& S'; \begin{array}[t]{@{}l@{}} (\href{../exec/runtime.html#syntax-init_table}{\mathsf{init\_table}}~\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}}~\mathit{eo}~\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}~\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}.\mathsf{init})^\ast \\ (\href{../exec/runtime.html#syntax-init_mem}{\mathsf{init\_mem}}~\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}}~\mathit{do}~\href{../syntax/modules.html#syntax-data}{\mathit{data}}.\mathsf{init})^\ast \\ (\href{../exec/runtime.html#syntax-init_global}{\mathsf{init\_global}}~\href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}~v)^\ast \\ (\href{../exec/runtime.html#syntax-invoke}{\mathsf{invoke}}~\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}})^? \\ \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}} \\ \end{array} \\ &\mbox{if} & \vdash \href{../syntax/modules.html#syntax-module}{\mathit{module}} : \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}^n \\ &\wedge& (\vdash \href{../exec/runtime.html#syntax-externval}{\mathit{externval}} : \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}')^n \\ &\wedge& (\vdash \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}}' \leq \href{../syntax/types.html#syntax-externtype}{\mathit{externtype}})^n \\[1ex] &\wedge& \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{globals} = \href{../syntax/modules.html#syntax-global}{\mathit{global}}^k \\ &\wedge& \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{elem} = \href{../syntax/modules.html#syntax-elem}{\mathit{elem}}^\ast \\ &\wedge& \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{data} = \href{../syntax/modules.html#syntax-data}{\mathit{data}}^\ast \\ &\wedge& \href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{start} = \href{../syntax/modules.html#syntax-start}{\mathit{start}}^? \\[1ex] &\wedge& S', \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}} = \mathrm{allocmodule}(S, \href{../syntax/modules.html#syntax-module}{\mathit{module}}, \href{../exec/runtime.html#syntax-externval}{\mathit{externval}}^n) \\ &\wedge& F = \{ \mathsf{module}~\href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}, \mathsf{locals}~\epsilon \} \\[1ex] &\wedge& (S'; F; \href{../syntax/modules.html#syntax-elem}{\mathit{elem}}.\mathsf{offset} \hookrightarrow^\ast S'; F; \href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~\mathit{eo})^\ast \\ &\wedge& (S'; F; \href{../syntax/modules.html#syntax-data}{\mathit{data}}.\mathsf{offset} \hookrightarrow^\ast S'; F; \href{../syntax/types.html#syntax-valtype}{\mathsf{i32}}.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~\mathit{do})^\ast \\ &\wedge& (S'; F; \href{../syntax/modules.html#syntax-global}{\mathit{global}}.\mathsf{init} \hookrightarrow^\ast S'; F; v)^\ast \\[1ex] &\wedge& (\href{../exec/runtime.html#syntax-tableaddr}{\mathit{tableaddr}} = \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{tables}[\href{../syntax/modules.html#syntax-elem}{\mathit{elem}}.\mathsf{table}])^\ast \\ &\wedge& (\href{../exec/runtime.html#syntax-memaddr}{\mathit{memaddr}} = \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{mems}[\href{../syntax/modules.html#syntax-data}{\mathit{data}}.\mathsf{mem}])^\ast \\ &\wedge& \href{../exec/runtime.html#syntax-globaladdr}{\mathit{globaladdr}}^\ast = \href{../exec/runtime.html#syntax-moduleinst}{\mathit{moduleinst}}.\mathsf{globals}[|\href{../syntax/types.html#syntax-externtype}{\mathrm{globals}}(\href{../syntax/modules.html#syntax-module}{\mathit{module}}.\mathsf{imports})|:k] Note

All failure conditions are checked before any observable mutation of the store takes place. Store mutation is not atomic; it happens in individual steps that may be interleaved with other threads.

Invocation¶

Once a module has been instantiated, any exported function can be invoked externally via its function address \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}\) in the store \(S\) and an appropriate list \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}^\ast\) of argument values. If successful, the invocation returns the function’s result values \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_{\mathrm{res}}^\ast\).

The following steps are performed:

  1. Assert: \(S.\mathsf{funcs}[\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}]\) exists.
  2. Let \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}\) be the function instance \(S.\mathsf{funcs}[\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}]\).
  3. Let \([t_1^n] \to [t_2^m]\) be the function type \(\href{../exec/runtime.html#syntax-funcinst}{\mathit{funcinst}}.\mathsf{type}\).
  4. If the length \(|\href{../exec/runtime.html#syntax-val}{\mathit{val}}^\ast|\) of the provided argument values is different from the number \(n\) of expected arguments, then:
    1. Fail.
  5. For each value type \(t_i\) in \(t_1^n\) and corresponding value \(val_i\) in \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}^\ast\), do:
    1. If \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_i\) is not \(t_i.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~c_i\) for some \(c_i\), then:
      1. Fail.
  6. Push the values \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}^\ast\) to the stack.
  7. Invoke the function instance at address \(\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}\).

Note

If the embedder performs type checks itself, either statically or dynamically, no failure can occur.

Once the function has returned, the following steps are executed:

  1. Assert: due to validation, \(m\) values are on the top of the stack.
  2. Pop \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_{\mathrm{res}}^m\) from the stack.

The values \(\href{../exec/runtime.html#syntax-val}{\mathit{val}}_{\mathrm{res}}^m\) are the results of the call.

If the function terminates with a trap, the error is propagated to the caller in a manner specified by the embedder.

\[\begin{split}\begin{array}{@{}lcl} \mathrm{invoke}(S, \href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}, \href{../exec/runtime.html#syntax-val}{\mathit{val}}^n) &=& \href{../exec/runtime.html#syntax-val}{\mathit{val}}_{\mathrm{res}}^m / \href{../exec/runtime.html#syntax-instr-admin}{\mathsf{trap}} \\ &\mbox{if}& S.\mathsf{funcs}[\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}].\mathsf{type} = [t_1^n] \to [t_2^m] \\ &\wedge& \href{../exec/runtime.html#syntax-val}{\mathit{val}}^n = (t_1.\href{../syntax/instructions.html#syntax-instr-numeric}{\mathsf{const}}~c)^n \\ &\wedge& S; \href{../exec/runtime.html#syntax-val}{\mathit{val}}^n~(\href{../exec/runtime.html#syntax-invoke}{\mathsf{invoke}}~\href{../exec/runtime.html#syntax-funcaddr}{\mathit{funcaddr}}) \hookrightarrow^\ast S'; \href{../exec/runtime.html#syntax-val}{\mathit{val}}_{\mathrm{res}}^m / \href{../exec/runtime.html#syntax-instr-admin}{\mathsf{trap}} \\ \end{array}\end{split}\]
